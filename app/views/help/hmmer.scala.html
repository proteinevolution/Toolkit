HMMER is a tool for fast and sensitive homology searching based on profile Hidden Markov models. It accepts a single sequence or a multiple alignment in aligned FASTA/CLUSTAL format as input. In case the input is a single sequence, HHblits is run over Uniprot20 to build an alignment required for the calculation of query HMM.
  • Input
    You have the option to enter or upload either a single protein sequence or a multiple sequence alignment (MSA) in FASTA or CLUSTAL format.

    Single query sequence:

    You can enter a single query sequence in FASTA format (with or without name line):

  • Select database
    The selected database is used by HMMER to perform the search.
    • nr50, nr70, nr90: NCBI's nonredundant protein sequence database filtered for a maximum pairwise sequence identity of 50%, 70% and 90%, respectively.
    • env70, env90: NCBI's environmental sequence database, usually from unknown organisms, filtered for a maximum pairwise sequence identity of 70% and 90%, respectively.
    • nr_euk*, nr_bac*, nr_arc*, nr_pro*, nr_vir*: eukaryotic, bacterial, archaeal, prokaryotic, and viral sequences from the nonredundant database nr.
    • pdb_nr: Sequences from proteins whose structures have been deposited in the Protein Data Bank PDB.
    • Uniprot_sprot: manually annotated and reviewed sequences from the UniProtKB.
    • Uniprot_trembl: automatically annotated and not reviewed sequences from the UniProtKB.
  • MSA enrichment iterations using HHblits
    Specifies the number of HHblits iterations over Uniprot20 to build an MSA when the input is a single sequence.
  • E-value
    report target sequences with an E-value of <= the specified value.
  • No. of target sequences
    This parameter controls how many matches will be displayed in the results.
